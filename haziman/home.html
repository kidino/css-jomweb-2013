HTML&CSS <IMG /> WITHOUT IMAGE?


by Haziman Hashim Aug 21st, 2013

 

 

Can you believe if I say that the picture above does not contain any jpg or gif or any other type of picture at all? Well believe it. Why? Because it doesn't. It is simply an arrangement of 1 pixel html SPAN element beside each other. While it is true that doing this might cause the page to consume more browser's processing resource. However, i think it is already a good start towards breaking the wall of implementing image element with no image resource. This idea came out when i was working on JOMWEB's second challenge - Cabaran CSS Layout where it requires all participation to not to use any image and javascript resources in a web layout. Pretty tough huh? But I have done this kind of thing since years ago, because i did not use any What You See Is What You Get (WYSIWYG) editors in that period like Adobe's ® Dreamweaver™ or so.

The Idea

As we all know, images are composed by arrangement of pixels. Each of these pixels holds their own color to form the image we saw. Applying the same concept into html, those pixel can be translated into html markup, then be styled so that they float next to each other, for each row. And then, displaying the resulting html within an IFRAME element (I would prefer doing this using DIV but the challenge's rule prohibits doing so).

Translating pixels

To generate the html markup for the image, i have wrote a ruby script which requires the rmagick gem (ImageMagick API for ruby language). So, assuming you have ruby and rubygems installed, you may use the script after executing the following command in the terminal:
gem install rmagick
After installing the gem. You can then use the following command to generate the image's HTML markup
./imagetohtml.rb <YourImageFileGoesHere.jpg>
If everything goes all right, a new file with the same name of the original file, with extension of ".html" will be created in the same directory. That file is the one that we will be using (still did not believe that it does not have any image file? go load it into any text editor).

Using the *.jpg.html file in our HTML

In order to be able to display the so-called "image" file, we will need to use the IFRAME element. Be advised that IFRAME element does not resize itself to fit the image size. Thus, we will need to define this as attribute of the IFRAME element like the following:
<iframe class="i2h-container" width="image width" height="image height" scrolling="no"></iframe>
And the resulting image would be similar to the first image in this article, depending on the width and height of the original image, and what image itself (of course).

Did anyone curious about the "i2h-container" class? The definition is like the following:
.i2h-container {

border: none;
overflow: hidden;
margin:0;
padding:0;
display:inline-block;
border-radius: 5px;

}

Known Issues

The only concerning issues is that the generator generates larger file than the original file. (due to repeating html tags - you know, html tags are so long). Currently, I'm still trying to piece together on how to solve this.
